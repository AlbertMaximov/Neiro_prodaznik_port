# Развертывание Нейро-Продажника (Docker Compose)

Проект полностью подготовлен для запуска через Docker Compose. В процессе сборки создается оптимизированный статический образ на базе Nginx.

Так как ключи API не должны "зашиваться" в публичные образы (built image), в проекте реализован механизм подмены плейсхолдеров. Это позволяет один раз собрать образ, а переменную окружения \`GEMINI_API_KEY\` передавать динамически при запуске контейнера.

## Как запустить

**Шаг 1. Создайте `.env` файл (для локального запуска):**

Продублируйте файл `.env.example` как `.env` и укажите в нём свой API-ключ:

```bash
cp .env.example .env
```
Убедитесь, что внутри \`.env\` прописано:
\`GEMINI_API_KEY=ваш_реальный_ключ_здесь\`

**Шаг 2. Запустите Docker Compose:**
В корне проекта выполните:

```bash
docker-compose up -d --build
```

**Шаг 3. Откройте приложение:**
Приложение будет доступно по адресу [http://localhost:8080](http://localhost:8080).

***

## Особенности сборки ("Под капотом")
1. В \`Dockerfile\` Vite-приложение собирается в статические файлы с "заглушкой" `__VITE_GEMINI_API_KEY_PLACEHOLDER__`.
2. При старте контейнера скрипт \`docker-entrypoint.sh\` автоматически находит эту заглушку в собранных `.js` файлах и заменяет её на значение переменной среды `$GEMINI_API_KEY`, переданной Docker'ом.
3. Раздачу статики обеспечивает быстрый легковесный `nginx:alpine`.
